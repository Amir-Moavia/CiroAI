# CrisisAI 🚨
> Real-time agentic crisis response system for Pakistan

CrisisAI is a high-fidelity mobile application that serves as an agentic real-time crisis response system designed specifically for Pakistan. It helps citizens, emergency responders, and city authorities by aggregating fragmented signals from social media, sensors, and reports to coordinate rapid, intelligent responses. Pakistan urgently needs this solution due to the increasing frequency of extreme weather events and infrastructure challenges that require immediate, automated coordination to save lives and resources.

---

## The problem
Fragmented crisis response in Pakistan often leads to delayed or ineffective emergency management. Urban flooding, severe heatwaves, and sudden road blockages frequently paralyze cities, while power outages and critical infrastructure failures compound the chaos. Currently, slow coordination between separate departments prevents a unified awareness of the situation. This lack of real-time, cross-correlated data directly threatens public safety and delays critical aid.

---

## The solution
CrisisAI addresses these challenges by deploying a 5-agent pipeline that observes, reasons, decides, acts, and adapts in real time. By autonomously processing multilingual reports and sensor data, the system identifies threshold breaches and correlates disparate signals. It calculates severity scores, creates prioritized action plans, and simulates response efficacy before execution. This ensures that response teams are deployed efficiently and citizens receive life-saving alerts without delay.

---

## Architecture — 5-agent pipeline

```text
INPUTS (Social Media / Sensors / Maps / Weather)
        ↓
[Agent 1: Input Agent]
 → Ingests multi-format signals
 → Normalises Urdu-Roman + English
 → Extracts location, type, severity
        ↓
[Agent 2: Detection Agent]
 → Clusters anomalies
 → Cross-correlates multi-source signals
 → Flags confirmed crisis with confidence %
        ↓
[Agent 3: Analysis Agent]
 → Scores severity (0-100)
 → Estimates cascade effects
 → Generates human-readable explanation
        ↓
[Agent 4: Planning Agent]
 → Prioritises actions (life safety first)
 → Allocates resources
 → Generates authority + citizen actions
        ↓
[Agent 5: Simulation Agent]
 → Executes actions (simulated)
 → Creates tickets + alerts
 → Calculates before/after metrics
        ↓
OUTPUT (React Native App — Map + Dashboard + Logs)
 → Live map markers
 → Crisis cards
 → Agent reasoning trace
 → Before/after impact panel
```

This system is truly agentic because the agents do not simply summarize data; they observe, reason, decide, act, evaluate, and adapt based on the environment. They maintain state, handle fallbacks automatically when sub-systems fail, and explain their decisions transparently.

## Assumptions

Data and scope:
- All crisis signals in the demo are synthetic/mock data
- Geographic focus is Islamabad, Pakistan (expandable to any city)
- Urdu support covers Roman-Urdu transliteration only, not full NLP
- Population density estimates are approximate, not census-accurate
- Sensor data (grid, water, telecom) is fully simulated

Technical assumptions:
- App is designed mobile-first (iOS and Android via Expo Go)
- Internet connection required for live map tiles
- Offline-first pipeline: all 5 agents run locally on device without requiring a backend server call
- Agent reasoning is rule-based with optional LLM enhancement via Groq or Gemini API (switchable in constants/config.ts)
- No real SMS gateway is connected — alerts are simulated
- React Native Maps falls back to a web stub on browser preview; real map only renders on physical device or emulator

Privacy:
- No user data is collected or stored externally
- All processing happens on-device
- No personal information appears in any mock dataset

## Demo path for judges

### How to see the full agentic system in 60 seconds

1. Open the app on your phone via Expo Go (scan QR code from terminal after `npx expo start`)

2. On the Home screen, tap the red "Demo Mode" button (bottom right corner, floating button)
   → This loads 8 pre-built Pakistan crisis signals
   → Watch all 5 agents complete in sequence on screen

3. Tap the Map tab
   → See crisis markers appear on live Islamabad map
   → Tap any red marker to see crisis details and confidence %
   → Red lines = blocked roads, Green dashed = alternate routes

4. Tap the Simulate tab
   → Select "F-7 Power Grid Failure" from dropdown
   → Tap "Run Simulation" button
   → Watch before/after metrics animate
   → See action execution feed appear in real time
   → Check the edge case: team delay + fallback activation

5. Tap the Logs tab
   → See full Antigravity agent reasoning trace
   → Each agent shows: observation → reasoning → decision → output
   → Tap any ⚠️ EDGE CASE entry to see failure + recovery
   → Tap "Baseline Comparison" tab to see agentic vs simple system

6. On the Alerts tab
   → See simulated SMS preview (what citizens would receive)
   → Check delivery stats: 12,450 notified, 98% delivered

Key things to look for as a judge:
- Agents observe AND reason — not just keyword matching
- Confidence scores change based on signal quality
- Cascading effects are inferred, not hardcoded
- Edge cases fail gracefully with logged recovery
- Full reasoning trace proves agentic behavior

## Cost and scalability

Per pipeline run (current):
- Agent 1 Input: ~50ms (local, no API)
- Agent 2 Detection: ~80ms (local)
- Agent 3 Analysis: ~120ms (local)
- Agent 4 Planning: ~90ms (local)
- Agent 5 Simulation: ~150ms (local)
- Total pipeline: ~490ms per crisis event
- With Groq/Gemini API: add ~300-800ms per LLM call
- Cost per LLM call (Groq): ~$0.0001 (near zero for demo)
- Cost per LLM call (Gemini Flash): ~$0.000075

Scaling discussion:
- 10x scale: run agents as async parallel workers, add Redis queue for signal ingestion
- 100x scale: split agents into microservices, use Kafka for event streaming, PostgreSQL for persistence
- Current demo handles: up to 50 concurrent signals cleanly
- Bottleneck at scale: map rendering + state updates (solvable with virtualized lists and marker clustering)

## Robustness evidence

1. Missing location → inferred from nearby signal cluster
2. Contradictory signals → crowd overrides stale API, logged
3. Resource unavailable → fallback team activated, ETA adjusted
4. Multi-crisis resource conflict → prioritised by severity score
5. Low signal quality → classified UNCONFIRMED, no false alert

Each case is selectable in the Simulate screen and fully visible in the Logs trace.

## Limitations

- Map markers are synthetic — not connected to real Pakistan crisis feeds
- Urdu support is transliteration dictionary, not trained NLP model
- No real utility API integration (WAPDA, CDA not publicly accessible)
- SMS alerts are simulated — no Twilio or SMS gateway connected
- Agent reasoning is deterministic rules + optional LLM, not fully autonomous
- Tested on Islamabad geography only — other cities need area data added
- Expo Go required for map — web preview shows stub only
